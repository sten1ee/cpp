  • Execuatbles
    • jacc.exe
    The main application of the distribution. It is a console application. Run with command line option -h to get a brief help. The default template file (jacc.tmpl) should be located in the same directory for the program to function properly. Here is the list of some important files:
    Jacc.ide
    The Borland C++ (v5.02) project file for Jacc
    jacc.tmpl
    The default template file used by Jacc
    jacc.y
    Bison specifications for the jacc file format
    lr.exe
    Jacc's ancestor. This program takes as input a simple grammar specification - no token type definitions, no semantic actions (see file erl.grm). It generates LR(1) table for the specified grammar and then reduces it to a LALR(1) table. Run with command line option `-h' to get a brief help. Run with command line `lalr -i erl.grm -o res.txt' and then view file `res.txt'
    calc.exe
    A simple example of Jacc generated parser - interactive command line calculator. Its input (taken from stdin) consists of series of expressions terminated with `;'. After each successive expression is entered followed by a semicolon, it is evaluated and the result is printed to stdout. The session is ended when EOF is reached or entering the special (pseudo EOF) symbol `$' . Here is the list of all calc related files:
    calc.out
    report file generated by running Jacc with command line: `jacc -v calc.j'
    calc.j
    the Jacc specifications of calc
    calc_main.cpp
    'hand made' file containing the main function and a simple scanner class
    calc_parser.h
    Jacc generated header. Contains the declaration of class calc_parser
    calc_parser.cpp
    Jacc generated source. Contains the implementation of class calc_parser
    calc_sym.h
    Jacc generated header. Contains the definitions of symbol codes and semantic value classes
    java12.exe
    A more sofosticated example of Jacc generated parser. This is a complete java1.2 parser. The distribution contains a java file: BitSet.java. You can run the command line java12 < BitSet.java (which is supposed to be a successful parse). Then edit BitSet.java screwing the syntax at some place and run the parser again. Note: this is just a parser ! It does not detect semantic errors !
    java12.j
    Jacc specification for the Java 1.2 parser
    java12.lex
    Flex lexical specifications for Java 1.2
    java12_lex.cpp
    Flex generated scanner for Java 1.2
    java12_parser.h
    Jacc generated header
    java12_parser.cpp
    Jacc generated source
    java12_sym.h
    Jacc generated header
